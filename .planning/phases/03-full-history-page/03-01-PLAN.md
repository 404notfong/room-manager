<task type="auto">
  <name>Task 1: Create TenantHistoryPage with filters and pagination</name>
  <files>frontend/src/pages/tenants/TenantHistoryPage.tsx</files>
  <action>
Create a new page component `TenantHistoryPage`:

1. **URL params**: Use `useParams()` to get `id` from route `/tenants/:id/history`.

2. **Fetch tenant**: `useQuery(['tenant', id], ...)` to get tenant name for the header.

3. **State**: 
   - `typeFilter: string` — 'all' | 'contract' | 'invoice' | 'payment'
   - `startDate: string` — date range start
   - `endDate: string` — date range end
   - `currentPage: number` — pagination page
   - `pageSize: number` — items per page (default 10)

4. **Fetch history**: `useQuery(['tenant-history', id, typeFilter, startDate, endDate, currentPage, pageSize], ...)` with all filter params.

5. **Layout**:
   ```
   [← Back to Tenants]  [Tenant Name — History]
   
   [Type Filter ▼]  [Start Date]  [End Date]
   
   [Timeline events with expandable cards]
   
   [Pagination]
   ```

6. **Header**: 
   - Back button (ArrowLeft icon) with `useNavigate()` to `/tenants`
   - Tenant fullName + " — " + t('tenants.history.title')

7. **Filters row**:
   - Type filter: `<Select>` with options All/Contract/Invoice/Payment
   - Start date: `<Input type="date">` 
   - End date: `<Input type="date">`
   - Filters update query params → triggers re-fetch

8. **Timeline**: Reuse the event rendering pattern from `TenantHistoryTimeline` but with expandable cards:
   - Click on an event card toggles expanded view
   - Collapsed: icon + title + date (one line)
   - Expanded: full details (room, rent, amount, status, etc.)

9. **Pagination**: Use existing `<Pagination>` component at bottom.

10. **Empty state**: Show message when no events match filters.

11. **Loading state**: Skeleton placeholders while fetching.
  </action>
