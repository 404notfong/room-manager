2. **Update tenants.service.ts**: 
   - Inject `Contract`, `Invoice`, `Payment` models in constructor:
     ```typescript
     constructor(
       @InjectModel(Tenant.name) private tenantModel: Model<TenantDocument>,
       @InjectModel(Contract.name) private contractModel: Model<ContractDocument>,
       @InjectModel(Invoice.name) private invoiceModel: Model<InvoiceDocument>,
       @InjectModel(Payment.name) private paymentModel: Model<PaymentDocument>,
     ) { }
     ```
   - Add `getHistory(tenantId: string, ownerId: string, query: GetTenantHistoryDto)` method:
     a. First verify tenant exists and belongs to owner (reuse findOne pattern)
     b. Build date filter: `{ createdAt: { $gte: startDate, $lte: endDate } }` if provided
     c. Build base filter: `{ tenantId: new Types.ObjectId(tenantId), isDeleted: false, ...dateFilter }`
     d. Based on `query.type` filter, query 1 or all 3 collections in parallel using `Promise.all`:
        - Contracts: `.find(filter).populate('roomId', 'roomName').sort({ createdAt: -1 })`
        - Invoices: `.find(filter).sort({ createdAt: -1 })`
        - Payments: `.find(invoiceFilter).populate('invoiceId', 'invoiceNumber').sort({ paymentDate: -1 })`
     e. Map each result to `TenantHistoryEvent`:
        - Contract → `{ type: 'contract', date: doc.createdAt, title: 'Contract created/terminated/etc', data: {...} }`
        - Invoice → `{ type: 'invoice', date: doc.createdAt, title: 'Invoice #XXX', data: {...} }`
        - Payment → `{ type: 'payment', date: doc.paymentDate, title: 'Payment received', data: {...} }`
     f. Merge all events, sort by date descending
     g. Apply pagination: slice array from `(page-1)*limit` to `page*limit`
     h. Return `{ data: events, meta: { total, page, limit, totalPages } }` matching existing findAll pattern

3. **Update tenants.controller.ts**: Add new endpoint AFTER the findOne method:
   ```typescript
   @Get(':id/history')
   getHistory(
     @Param('id') id: string,
     @CurrentUser() user: any,
     @Query() query: GetTenantHistoryDto
   ) {
     return this.tenantsService.getHistory(id, user.userId, query);
   }
   ```
   IMPORTANT: Place this route BEFORE the `@Get(':id')` route, or NestJS will match `:id` = "history". Actually, since `:id` is before `/history` path-wise, put `@Get(':id/history')` AFTER `@Get(':id')` — NestJS will match the more specific route first. But to be safe, verify the route ordering.
